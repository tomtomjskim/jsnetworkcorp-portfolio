# Portfolio Screen Spec

Each portfolio screen should include:

- purpose
- target user
- entry point
- main components
- visible states
- validation rules
- API or data relationship
- publication status
- AI or agent usage, when relevant
- verification evidence

Screen status labels:

```text
implemented | prototype | planned | sanitized | archived
```

## Done Criteria

- One-line project summary exists.
- At least three screen specs exist for a major project.
- At least one wireframe or screenshot placeholder exists.
- Architecture diagram is present or marked pending.
- Verification evidence exists or is marked pending.
- Publication boundary was reviewed.
