# Resume Data Changelog

This changelog tracks the public-safe resume data model. Detailed raw evidence and private analysis remain outside this public repository.

## 2026-06-29 / RB-v0.5 / PF-v0.5.0

### Added

- Introduced a sanitized resume data layer under `docs/resume-data/`.
- Added public-safe claim management for backend/full-stack resume positioning.
- Added a sanitized commerce/fulfillment project one-pager under `content/projects/`.
- Added a Korean public-safe resume draft under `content/resume/`.
- Added explicit redaction rules for public portfolio content.

### Changed

- Promoted commerce/logistics operations experience to a major portfolio axis.
- Reframed the commerce project as an anonymized seller commerce and fulfillment operations system.
- Reframed live-commerce as a portfolio-adjacent commerce-channel expansion theme, not a completed integration claim.
- Kept private source evidence and detailed claim-to-evidence mapping outside this public repository.

### Removed / Excluded

- Removed unverifiable quantitative improvement claims from public resume usage.
- Excluded unfinished/non-commercial portfolio projects from primary resume positioning.
- Excluded private repository names, private source links, internal specs, raw evidence, staging configuration, credentials, order/customer/admin/session/payment data, and raw AI transcripts.

## 2026-06-29 / RB-v0.4

### Added

- Consolidated the company commerce/logistics project into the resume raw data model.
- Identified public-safe themes: seller operations, external order registration, inbound/inventory/outbound workflows, fulfillment integration, idempotency, masked payloads, fail-closed gates, and sync verification.
- Added a role-scope confirmation requirement before using strong verbs such as designed, implemented, or owned.

### Decision

Use anonymized, domain-oriented descriptions in public materials. Keep direct project identifiers and private evidence in restricted storage only.

## 2026-06-29 / RB-v0.3

Treat the project as company/private evidence, not as a public portfolio repository link.
